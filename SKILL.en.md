# E-commerce Visual Planning and Execution SOP

## Language Rule

The default runtime entry is `SKILL.md`, which outputs Chinese by default. Use this English file when the user explicitly asks for English or a bilingual version.

## When To Use

Use this skill when the user needs product main images, detail pages, selling-point visualization, on-image copy, detail-page narrative, platform compliance checks, or image-generation prompts.

Typical triggers:

- "Create Taobao main images and a detail page for this product."
- "Turn these product selling points into main-image copy."
- "Give me a Douyin Shop visual storyboard."
- "Check whether this detail page has ad-law risks."
- "Give me the main-image storyboard first, then write the on-image copy."
- "My designer needs scene direction + on-image copy + design notes."

Do not use it for:

- Final legal advice or a replacement for platform/legal review
- Fabricated test reports, patents, approval numbers, sales, reviews, or certifications
- Medical, drug, health-food, or device claims without qualified evidence
- Automatic listing, publishing, ad launching, or external release
- Generic marketing long-form copy unrelated to e-commerce visual execution

## Core Principles

1. Strategy first. Do not design scenes before identifying the conversion driver.
2. Main images and detail pages serve different goals. Main images optimize CTR; detail pages optimize CVR.
3. Storyboard first, final copy second. Do not skip the direction gate.
4. Use legal baseline first, then add platform-specific constraints.
5. Five independent review dimensions. If any dimension scores below 80, rewrite that dimension instead of averaging it away.

## Workflow

1. Collect information and mark missing items.
2. Identify the conversion driver and lock the Campaign Style Lock.
3. Output the Visual Strategy Brief and pause for user confirmation.
4. Output the Image Storyboard and pause for user confirmation.
5. After confirmation, output scene direction, on-image copy, design notes, and prompts.
6. Run five independent self-review checks; rewrite any dimension below 80.
7. Deliver the final execution plan and risk notes.

## Step 1: Deep Planning

Collect the following information first. If anything is missing, mark it as "missing" and ask only the questions that affect direction or compliance.

| Required item | Purpose |
|---|---|
| Product type and qualification boundary | Determines compliance red lines |
| Target platform | Adds platform review constraints |
| Target audience and usage scenarios | Grounds pain points and purchase reasons |
| Selling points, specs, price, or SKU | Builds Feature -> Benefit translation |
| Existing images, packaging, detail page, or competitor references | Determines the visual starting point |
| Visual style references | Builds the Campaign Style Lock |

Conversion driver types:

| Type | Typical categories | Visual focus |
|---|---|---|
| Visual-driven | Beauty, jewelry, decor | Texture, lighting, packaging, atmosphere |
| Pain-point-driven | Cleaning, storage, baby products, care goods | Contrast, proof, safety, problem-first framing |
| Rational-driven | Digital, appliances, 3C accessories | Parameter translation, mechanism visualization, quantified differences |
| Emotional-value-driven | Fragrance, gift boxes, pet goods, holiday products | Mood, ritual, after-use state |

Campaign Style Lock must include:

- Primary, secondary, and accent colors, no more than three colors
- Product area ratio, usually 60%-80% for main images
- Lighting system
- Typography system
- Drift bans, such as random backgrounds, fake logos, style jumps, or excess decoration

## Step 2: Visual Strategy Brief

Output the following, then pause for confirmation:

1. Conversion driver and evidence
2. Campaign Style Lock
3. Feature -> Advantage -> Benefit -> Evidence table
4. Compliance boundary: likely legal and platform red lines
5. Missing information and current assumptions

Pause wording:

> This is the visual strategy brief. Please confirm whether the conversion-driver judgment and style lock fit the brand tone; after confirmation, I will continue with the main-image and detail-page storyboard.

## Step 3: Image Storyboard

This step outputs direction only, not final on-image copy. The goal is to expose direction errors before expensive image execution.

### Five Main Images

| Item | Task | Script notes |
|---|---|---|
| KV1 Hero | Visual interception | Product takes 60%-80%; one core selling point; clear first visual focus |
| KV2 Pain point | Problem-first | Use customer language; avoid feature dumping |
| KV3 Difference | Visualized difference | Before/after, old/new, or parameter comparison; let the visual prove it |
| KV4 Scenario | Mirror effect | Real usage context without burying the product |
| KV5 Decision | Purchase action | Specs, price, bundle, user type, or CTA; numbers before adjectives |

Each main-image script must include:

- Visual task
- Composition notes: product ratio, angle, eye-flow
- First visual focus
- Color restraint
- Banned expressions

### Detail Page Narrative

The detail page optimizes conversion. Users scroll with a skeptical mindset, so the sequence must follow "attention -> desire -> trust -> action."

| Module | Narrative job | Script notes |
|---|---|---|
| M1 Opening | Make the user feel "this is for me" within 3 seconds | Pain point or strong hook, not a plain product shot |
| M2-M3 Core benefits | Expand the top 2-3 purchase reasons | Translate Features into Benefits |
| M4 Material/craft | Haptic expression | Define hard, soft, translucent, crisp, warm, or other tactile cues and lighting |
| M5 Scenarios | Real use cases | 2-3 concrete scenarios |
| M6 Proof | Visualized proof | Test reports, patents, and credentials only when provided by the user |
| M7 Comparison | Reduce comparison anxiety | Objective parameter comparison, no competitor disparagement |
| M8 FAQ | Defend against objections | Questions derived from user concerns |
| M9 CTA/statement | Action and risk reversal | Price, specs, guarantee, and statement placement |

Pause wording:

> This is the image skeleton, not final copy. Please confirm whether the main-image task split and detail-page sequence are right; after confirmation, I will produce the final scenes, copy, and design notes.

## Step 4: Execution Output

After storyboard confirmation, deliver four items for each image:

1. Scene direction: layout, product ratio, lighting, background, and eye-flow.
2. On-image copy: benefit-translated copy; main images up to 5 lines, each line up to 6 Chinese characters or equivalently short English; detail modules up to 6 lines.
3. Design notes: first visual focus, eye-flow, color psychology, disclaimer placement.
4. Image-generation prompt: Campaign Style Lock, subject, composition, material, lighting, background, and restrictions.

## Step 5: Five-Part Independent Self Review

Score each dimension from 0 to 100 independently. If any dimension is below 80, rewrite that dimension and score again.

| Dimension | Basis | Zero or heavy penalty |
|---|---|---|
| Compliance | Legal baseline first, platform rules second | Absolute claims, fabricated proof, medicalized claims, food-function claims |
| Benefit translation | Whether Features are translated into user-perceived Benefits | Manual-style parameter dumping or function-only copy |
| First visual focus | Product ratio, single selling point, copy line count, eye-flow | Small product, stacked claims, overlong copy, unclear focus |
| Haptic/media expression | Material lighting and separation of main-image vs detail-page logic | No tactile description, mixed CTR/CVR logic, style drift |
| Narrative continuity | Non-repetitive five-image task split and conversion-path detail page | Repeated information, jumpy modules, missing proof |

Detailed rules live in `references/compliance-rules.md`.

## Missing-Information Rules

If inputs are incomplete:

- Do not fabricate qualifications, tests, sales, reviews, or brand endorsements.
- Output a "missing-information strategy brief" and mark assumptions.
- For high-risk categories, output only neutral visual directions and safe copy until qualifications are supplied.
- Ask at most three key questions, prioritizing those that change positioning or compliance.

## Key Failure Modes

- User asks for scenes before conversion-driver judgment: return to Step 1.
- User wants to skip storyboard: explain that it prevents expensive rework; pause unless they explicitly say "use your judgment and continue."
- Five main images repeat the same information: rewrite the whole image task split.
- Detail modules are stacked randomly: reorder along the conversion path.
- Compliance review uses keyword guessing only: explain the risk category and evidence gap, but do not give final legal advice.
- User asks for stronger or more absolute claims: strengthen visual impact, not compliance risk.
