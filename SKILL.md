# 电商视觉策划与执行 SOP

## 语言规则

默认使用中文输出。用户明确要求 English、英文版或 bilingual 时，切换到英文或中英双语输出。英文完整版本见 `SKILL.en.md`，但运行时仍以本文件作为默认入口。

## 什么时候使用

当用户要做商品主图、详情页、卖点视觉化、图内文案、详情页叙事、平台合规自查或生图 Prompt 时使用本 Skill。

典型触发：

- "帮我做一套淘宝主图和详情页文案"
- "这个产品怎么提炼卖点做主图"
- "按抖音小店风格给我出视觉脚本"
- "帮我检查这套详情页有没有广告法风险"
- "先给我主图 storyboard，再写图内文案"
- "设计师要开工，给我画面 + 图内文案 + 设计说明"

不要用于：

- 给出最终法律意见或替代律师/平台审核结论
- 编造检测报告、专利、批准文号、销量、评价、认证
- 为医疗、药品、保健食品、器械类产品制造未经资质支持的功效背书
- 自动上架、自动投放、自动对外发布
- 和电商视觉落图无关的普通营销长文

## 核心原则

1. 策划先行。未判断转化驱动力前，不进入具体画面设计。
2. 主图和详情页分工不同。主图目标是点击率，详情页目标是转化率。
3. 先 Storyboard，后最终文案。方向确认前不做重执行。
4. 合规底线先按法律基线判断，再叠加平台规则。
5. 五维独立自审。任一维度低于 80 分，重写该维度，不用其他高分抵消。

## 工作流

1. 收集信息，标注缺失项。
2. 判断转化驱动力，锁定 Campaign Style Lock。
3. 输出《视觉策划案》，暂停等待用户确认。
4. 输出《图片规划脚本 / Storyboard》，暂停等待用户确认。
5. 用户确认后，输出画面、图内文案、设计说明和生图 Prompt。
6. 做五维独立自审，任一维度低于 80 分则只重写该维度。
7. 交付最终执行方案和风险提示。

## Step 1：深度策划

先收集以下信息。缺项时不要编造，直接标注"缺失"，并只追问会影响方向判断的关键项。

| 必须项 | 用途 |
|---|---|
| 产品类型与资质边界 | 判断适用合规红线 |
| 目标平台 | 叠加平台审核规则 |
| 目标人群与使用场景 | 提炼痛点、场景和购买理由 |
| 产品卖点、参数、价格或规格 | 建立 Feature 到 Benefit 的翻译 |
| 已有主图、详情页、包装或竞品参考 | 判断视觉起点 |
| 视觉参考风格 | 建立 Campaign Style Lock |

转化驱动力判断：

| 类型 | 典型品类 | 视觉重点 |
|---|---|---|
| 视觉驱动型 | 美妆、饰品、家居摆件 | 质感、光影、包装、氛围 |
| 痛点驱动型 | 清洁、收纳、母婴、健康护理 | 对比、证据、安全感、问题先行 |
| 理性驱动型 | 数码、家电、3C 配件 | 参数翻译、机制可视化、数字量化 |
| 情感价值型 | 香氛、礼盒、宠物用品、节日商品 | 情绪、仪式感、使用后状态 |

Campaign Style Lock 必须包含：

- 主色、辅助色、强调色，不超过三色
- 产品画面占比，主图建议 60%-80%
- 光影体系
- 字体系统
- 禁止漂移项，例如随机背景、虚构 logo、风格突变、过度装饰

## Step 2：输出《视觉策划案》

输出以下内容后暂停，等待用户确认：

1. 转化驱动力判断与依据
2. Campaign Style Lock
3. 利益翻译表：Feature -> Advantage -> Benefit -> Evidence
4. 合规边界：本产品可能适用的法律和平台红线
5. 缺失信息与当前假设

暂停句：

> 以上是本次视觉策划案。请确认转化驱动力判断和风格锁是否符合品牌调性；确认后我再继续输出主图和详情页的视觉规划脚本。

## Step 3：输出《图片规划脚本 / Storyboard》

这一步只出导演脚本，不出最终图内文案。目标是提前暴露方向错误，避免在成图阶段返工。

### 主图 5 张任务分配

| 编号 | 任务 | 脚本要点 |
|---|---|---|
| KV1 首图 | 视觉截流 | 产品主体占比 60%-80%；唯一核心卖点；视觉第一落点明确 |
| KV2 痛点/问题 | 问题先行 | 用用户语言提出问题，不做功能堆砌 |
| KV3 差异对比 | 差异可视化 | 前后对比、新旧对比或参数对比，画面自证 |
| KV4 场景代入 | 镜像效应 | 产品在真实场景中使用，环境不能淹没主体 |
| KV5 行动引导 | 促成决策 | 规格、价格、组合、适用人群或 CTA，数字优先于形容词 |

每张主图脚本必须包含：

- 视觉任务
- 构图建议：主体占比、视角、视线流向
- 视觉第一落点
- 色彩克制方式
- 禁止出现的表达

### 详情页叙事流

详情页目标是转化率。用户带着证伪心态往下滑，模块必须按"吸引注意 -> 激发欲望 -> 建立信任 -> 推动行动"推进。

| 模块 | 叙事功能 | 脚本要点 |
|---|---|---|
| M1 首屏承接 | 3 秒内让用户知道"这是给我看的" | 痛点或强钩子，不是纯产品图 |
| M2-M3 核心获益 | 展开 Top 2-3 必卖理由 | Feature 必须翻译为 Benefit |
| M4 材质/工艺 | 触感表达 | 明确冷硬、柔软、透亮、酥脆等触感及对应光影 |
| M5 场景覆盖 | 代入真实使用 | 2-3 个具体场景 |
| M6 信任证据 | 证据可视化 | 检测报告、专利、资质只可基于用户提供资料 |
| M7 对比选择 | 消除比较焦虑 | 客观参数对比，不贬低竞品 |
| M8 FAQ | 主动防御顾虑 | 从用户顾虑反推问答 |
| M9 CTA/声明 | 推动行动和风险逆转 | 价格、规格、保障、声明位置明确 |

暂停句：

> 这是整套图的骨架规划，还未涉及最终文案。请确认主图的视觉任务分配和详情页的叙事顺序是否合理；确认后我再进入最终画面、文案和设计说明的输出。

## Step 4：执行输出

用户确认 Storyboard 后，每张图交付四项：

1. 画面描述：元素排布、主体占比、光影、背景、视线动线。
2. 图内文案：完成利益翻译；主图不超过 5 行、单行不超过 6 字；详情页模块不超过 6 行。
3. 设计说明：视觉第一落点、视线引导、色彩心理、免责声明位置。
4. 生图 Prompt：包含 Campaign Style Lock、主体、构图、材质、光影、背景、限制项。

输出格式：

```markdown
# 电商视觉策划与执行方案

## 1. 策划案回顾
- 转化驱动力：
- Campaign Style Lock：
- 合规边界：
- 当前缺失信息：

## 2. 主图 5 张
### KV1：...
- 画面：
- 图内文案：
- 设计说明：
- 生图 Prompt：

## 3. 详情页模块
### M1：...
- 画面：
- 图内文案：
- 设计说明：
- 生图 Prompt：

## 4. 五维自审评分

## 5. 风险提示与待补充信息
```

## Step 5：五维独立自审

每个维度 0-100 分独立判定。任一维度低于 80 分，必须重写该维度后重新评分。

| 维度 | 判定依据 | 0 分或重扣条件 |
|---|---|---|
| 合规性 | 法律基线逐条核查，平台规则作为附加约束 | 绝对化用语、虚构证据、医疗化暗示、食品功效暗示等命中红线 |
| 利益翻译度 | Feature 是否翻译为用户能感知的 Benefit | 残留说明书语言、参数不解释、只讲功能不讲好处 |
| 视觉第一落点 | 主体占比、卖点唯一性、文案行数、视线动线 | 主体过小、卖点堆砌、文案超行、第一落点混乱 |
| 触感/媒介表达 | 材质光影是否匹配产品属性，主图和详情页分工是否清楚 | 缺触感描述、主图详情页逻辑混用、风格漂移 |
| 叙事连贯性 | 主图 5 张是否去重，详情页是否沿成交路线推进 | 信息重复、模块顺序跳跃、信任证据缺失 |

合规分层提示：

| 品类 | 核心红线 |
|---|---|
| 所有产品 | 禁止虚假、引人误解、绝对化用语；数据宣称必须有来源 |
| 保健食品 | 只能基于批准文号和用户提供材料表达，不扩大功效 |
| 运动器材/体态管理 | 避免医疗术语、治疗动词和保证性效果 |
| 普通食品 | 不写疾病预防、治疗、身体改善或保健暗示 |

详细规则见 `references/compliance-rules.md`。

## 缺资料处理规则

如果资料不足：

- 不编造资质、检测、销量、评价、品牌背书。
- 可先输出"缺资料版策划案"，并标注哪些内容是待确认假设。
- 对合规高风险品类，缺资质时只输出视觉方向和中性文案，不输出功效承诺。
- 最多追问 3 个关键问题，优先问会改变定位或合规边界的问题。

## 关键失败模式

- 未先判断转化驱动力就直接生成画面：退回 Step 1。
- 用户要求跳过 Storyboard：提醒这是防返工门禁；除非用户明确说"按你判断直接出最终稿"，否则暂停确认。
- 主图 5 张信息重复：整组打回，重新分配视觉任务。
- 详情页模块随意堆叠：按成交路线重排。
- 合规判断只凭敏感词：必须说明对应风险类型和依据，不给最终法律意见。
- 用户要求"更猛、更绝对"：加强视觉冲击，不突破合规边界。
- 用户要求英文输出：可切换英文，但中文仍是默认输出语言。

## 交付前检查清单

- [ ] 是否明确产品类型、平台和资质边界？
- [ ] 是否判断了转化驱动力？
- [ ] 是否给出 Campaign Style Lock？
- [ ] 是否先输出策划案并暂停？
- [ ] 是否先输出 Storyboard 并暂停？
- [ ] 是否逐图给出画面、文案、设计说明和 Prompt？
- [ ] 是否完成五维独立自审？
- [ ] 是否标注缺失信息和风险？
- [ ] 是否避免编造资质、数据、销量、评价和功效？
